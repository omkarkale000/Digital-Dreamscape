💻 WSL2 + Ubuntu: Install Linux on Windows Without VirtualBox

🚀 Want to run Ubuntu directly on Windows? Here’s your VirtualBox-free alternative — fast, lightweight, and great for developers and DevOps learners.

📌 What is WSL?

Windows Subsystem for Linux (WSL) lets you run a GNU/Linux environment directly on Windows — no dual-boot, no heavy VM. WSL2 includes a real Linux kernel and provides better compatibility and performance than WSL1.

✅ Advantages

  • ⚡ Faster and more lightweight than most full VMs
  • 🛠 Full Linux kernel support in WSL2
  • 📦 Run Linux tools and Windows apps side-by-side
  • 🗂 Easy file sharing between Windows and Linux
  • 🎯 Great for development, scripting, and running Ansible

❌ Disadvantages (and quick workarounds)

  • 🌐 Dynamic IP/hostname — add `/etc/hosts` entries or use dynamic inventory (we include tips below).
  • 🔧 Kernel updates may be required — run wsl --update (covered in setup).
  • 🖥 No GUI by default — use WSLg for GUI apps or use a full Ubuntu desktop when needed.

🖥 System Compatibility

Minimum: Ryzen 3 / Intel i3, 4 GB RAM

Recommended: Ryzen 3 / Intel i3, 8 GB RAM

⚙️ Part 1 — Install WSL2

There are multiple ways to install WSL2. Choose the one you prefer.

Option A — Quick (Command line)

Runs everything automatically (requires Windows with WSL support):

wsl --install

Option B — Microsoft Store (GUI)

Open the Microsoft Store, search Windows Subsystem for Linux or the distro you want (e.g., Ubuntu) and install via the Store. This is the easiest GUI route.

SS/GIF placeholder: how to install from Microsoft Store

Option C — Manual (advanced)

If you prefer to enable features manually (PowerShell, Admin):

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
# Then restart your PC

🔧 Part 2 — Kernel Update

If the installer prompts you to update the WSL2 kernel (error like 0x800701bc), update with:

wsl --update

If the automatic update still complains, download the official kernel package from Microsoft: aka.ms/wsl2kernel — run the MSI, reboot, then continue.

Note: the GitHub WSL2 kernel repo has build instructions for advanced users — you don't need to compile anything for normal installs.

🐧 Part 4 — Install Ubuntu (or another distro)

WSL2 supports many distros. Popular choices:

  • Ubuntu
  • Debian
  • Kali Linux
  • openSUSE
  • AlmaLinux / Oracle Linux

Install Ubuntu 22.04:

wsl --install -d Ubuntu-22.04

After install, launch Ubuntu from the Start Menu, create your UNIX username & password and run the initial updates below.

🧰 Part 5 — Initial Ubuntu setup

Inside your new Ubuntu shell, run:

sudo apt update && sudo apt upgrade -y

Install basic tools (optional):

sudo apt install -y curl wget git nano build-essential

1️⃣ Change the Hostname

By default, WSL2 assigns a generic hostname. You can personalize it:

sudo hostnamectl set-hostname WSL2UB22OMK

Verify the change:

hostnamectl

2️⃣ Create a Root User

You can create a dedicated root user for administrative tasks:

sudo adduser rootuser

Grant root privileges:

sudo usermod -aG sudo rootuser
Note: Choose a strong password and store it securely. You will not be able to recover it if forgotten.

💡 Tips & Networking

Check host (Windows) network

ipconfig /all

Check Ubuntu network

ip addr
ping -c 3 google.com

If you need stable hostnames for Ansible testing, add the current WSL IP and a friendly name to /etc/hosts (on the control machine or targets) — or use a dynamic inventory script to regenerate the inventory at runtime.

📋 Useful Commands

Quick reference 
          
wsl --list --verbose            # list installed distributions
wsl --set-version Ubuntu 2      # set Ubuntu to WSL2
wsl --terminate Ubuntu          # stop Ubuntu session
wsl --unregister Ubuntu         # remove Ubuntu
sudo apt install       # install software
ls -la                          # list files with details
pwd                             # show current directory
top / htop                      # view running processes
df -h                           # disk usage
free -m                         # memory usage
uname -r                        # kernel version
cat /etc/os-release             # OS info
nano                      # edit file
ping google.com                 # test network
history                         # recent commands
grep                # search text
chmod +x                  # make executable
tar -czvf file.tar.gz folder/    # compress
scp file user@host:/path         # copy file over SSH
exit                            # close WSL session

🔍 WSL2 Ubuntu vs Full Ubuntu Desktop

  • WSL2 Ubuntu — lightweight, CLI-first, great for development and automation tasks. GUI possible via WSLg.
  • Full Ubuntu Desktop — full desktop experience, needs separate boot or a VM like VirtualBox/VMware.

🎯 Wrap up

WSL2 + Ubuntu gives you a fast, developer-friendly Linux environment on Windows. It’s ideal for Ansible labs, Docker tooling, scripting, and everyday development. Keep your kernel up-to-date and use host mapping or dynamic inventory to handle IP changes.
